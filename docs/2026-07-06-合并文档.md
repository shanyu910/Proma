## 一、项目简介

Legis 是从 Proma（一个开源的本地优先 AI 桌面应用）fork 出来的定制版本。在保留 Proma 原有 Chat + Agent 能力的基础上，做了以下定制：

1. **品牌替换**：Proma → Legis（包名、图标、文案、链接全部替换）
2. **登录认证**：对接 Legis-Server，软关卡 + 弹窗登录
3. **模型一体化**：只使用 AgentSkill 一个供应商，通过 SK 直连调模型
4. **权限管控**：未登录降级为只读浏览模式

## 五、认证架构详解

### 5.1 认证策略：软关卡

```
App 启动
  ↓
不管有没有登录，都能进主界面
  ↓
用户可以浏览历史聊天记录、工作区文件
  ↓
当用户想做"需要权限"的事时（发送消息/选模型/进设置）：
  → 弹出登录弹窗
  → 登录成功后继续操作
```

### 5.2 认证状态

```typescript
type AuthStatus = 'loading' | 'authenticated' | 'guest'
```

| 状态 | 含义 | 用户能做什么 |
|---|---|---|
| `loading` | App 启动时验证 Token 中 | 主界面可见，操作会等待 |
| `authenticated` | 已登录（有有效 Token） | 所有功能正常 |
| `guest` | 未登录 | 浏览历史、看文件；发消息/选模型/进设置会弹登录 |

### 5.3 数据流

```
启动：
  AuthInitializer 从 Keychain 读 Token
  ├─ 无 Token → authStatus = 'guest'
  └─ 有 Token → GET /auth/me 验证
       ├─ 有效 → authStatus = 'authenticated'
       │         → GET /auth/me/model-config → SK 存主进程内存
       │         → 同步到 channels.json（官方渠道）
       │         → GET /auth/me/model-usage → 余额存 atom
       └─ 无效 → 清 Keychain → authStatus = 'guest'

登录（弹窗）：
  POST /auth/login → Token 存 Keychain
  → authStatus = 'authenticated'
  → 拉取 model-config + model-usage
  → 执行 onSuccess 回调（如发送消息）

退出登录：
  清 Keychain + 清主进程 SK 内存
  → authStatus = 'guest'
  → 用户留在主界面（不跳转）
```

### 5.4 SK 完整数据流

```
登录成功（渲染进程）
  ↓ fetchModelConfigData(token)
  拿到 provider.apiKey（SK）
  ↓ IPC: legis:set-sk(SK)
  传到主进程 skInMemory（模块级私有变量）
  ↓
syncModelConfigToChannels(config)
  → upsertOfficialChannel（apiKey 写占位符 __LEGIS_INJECT__）
  → 禁用旧渠道
  → agentChannelId 切到 legis-official
  ↓
用户对话发请求（主进程）
  → decryptApiKey('legis-official')
  → channel.apiKey 解密后是 '__LEGIS_INJECT__'
  → 返回主进程 getSKInMemory()
  → 用真 SK 调 AgentSkill /v1/messages
```

## 六、如何同步上游 Proma 更新

### 6.1 前置条件


```bash
# 1. 拉取上游最新
git fetch upstream

# 2. 切到 main，合并上游 main（保持 main 干净）
git checkout main

git push origin main

# 3. 切到 Legis，合并 main
git checkout Legis
git merge main
# 此时可能有冲突，见下方冲突解决指南
```

### 6.3 冲突解决指南

合并后**最容易冲突的文件**及解决策略：

#### `renderer/App.tsx`（🔴 高风险）

**你改了什么**：加了 `<AuthInitializer/>` 和 `<LoginModal/>`，移除了旧硬关卡。

**冲突时**：
- 保留你的 `import { AuthInitializer, LoginModal } from '../legis'`
- 保留你的 `<AuthInitializer/>` 和 `<LoginModal/>` 挂载
- 合入上游对 AppShell/onboarding 逻辑的改动
- ⚠️ **注意 Hooks 顺序**：所有 `useEffect` 必须在条件 `return` 之前

#### `app-shell/LeftSidebar.tsx`（🔴 高风险）

**你改了什么**：底部根据登录态切换"登录"按钮/AccountMenu，设置入口加认证拦截。

**冲突时**：
- 保留你的 `import { useAuthGate, authStatusAtom, authUserAtom, AccountMenu }`
- 保留你的 `handleOpenSettings` 认证拦截逻辑
- 保留你的底部 `isGuest ? 登录按钮 : AccountMenu` 条件渲染
- 合入上游对侧边栏列表、样式等的改动

#### `chat/ChatInput.tsx` + `agent/AgentView.tsx`（🔴 高风险）

**你改了什么**：加了 `useAuthGate`，未登录禁用输入。

**冲突时**：
- 保留你的 `import { useAuthGate } from '../../../legis'`
- 保留你的 `const { isGuest, requireAuth } = useAuthGate()`
- 保留 RichTextInput 的 `disabled={isGuest}`
- 保留 handleSend 的 `requireAuth` 包裹
- 合入上游对发送逻辑的改动

#### `chat/ModelSelector.tsx`（🟡 中风险）

**你改了什么**：加了 `useAuthGate`，未登录显示灰色禁用按钮。

**冲突时**：
- 保留你的 `import { useAuthGate } from '../../../legis'`
- 保留你的 `isGuest` 判断和禁用 UI
- 合入上游对模型列表渲染的改动

#### `main/lib/channel-manager.ts`（🟡 中风险）

**你改了什么**：加了 `upsertOfficialChannel()`、`updateOfficialChannelModelSelection()`、`decryptApiKey` 占位符检测。

**冲突时**：
- 保留你末尾新增的 `upsertOfficialChannel` 等函数
- 保留 `decryptApiKey` 里的占位符检测逻辑
- 合入上游对渠道管理逻辑的改动

#### `main/ipc.ts` + `preload/index.ts`（🟡 中风险）

**你改了什么**：追加了 Token/SK/渠道/模型勾选的 IPC 通道。

**冲突时**：
- 你的 IPC 都是追加在末尾的，通常不会和上游改动冲突
- 如果冲突，保留你的 IPC handler，合入上游新增的 handler

#### `@proma → @legis` 的 import 替换

上游新增的文件用 `@proma`，合并后需要批量改：
```bash

bun run typecheck
```
